Array vs. Hash Showdown

11/14/2014

What is an array?

An object whose job is to store an ordered collection of objects. It looks like this:

food = Array.new
food = ["pizza", "steak", "soup"] or
numbers = Array.new
numbers = [1,2,6,4]

What is a hash?

Also an ordered collection of objects!
It looks like this:
food = Hash.new
food = {
"pizza" => "pepperoni",
"steak" => "rare",
"soup" => "vegetable"
}

You can also write it like this:

food = Hash.new
food = {
:pizza => "pepperoni",
:steak => "rare",
:soup => "vegetable"
}

Or even like this:

food = Hash.new
food[:pizza] = "pepperoni"
food[:steak] = "rare"
food[:soup] = "vegetable"

Ah! So what's the difference?

Hashes store their objects in pairs called keys and values and the order of the keys is crucial.

How do you access a value with an array?

food = ["pizza", "steak", "soup"]
food[0] = "pizza"
food[2] = "soup"

How do you access a value with a hash?

food = {
:pizza => "pepperoni"
:steak => "rare"
:soup => "vegetable"
}

food["pizza"] = "pepperoni"
food["soup"] = "vegetable"

Key things to remember:

  • If you need to store a specific value with a label/key, go with a hash!
  • To access an array, refer to where it's indexed (starts at 0).
  • To access a hash, refer to the key.
